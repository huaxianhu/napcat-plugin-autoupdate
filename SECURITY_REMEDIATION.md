# 安全处置报告：移除疑似后门/恶意行为

## 背景

在审计中发现 `src/auto-sign.ts` 存在与“插件自动更新”目标无关的高风险逻辑，并且会在插件初始化时自动启动。

## 已确认的问题

1. 代码内置外部接口 `https://i.elaina.vin/api/openai/qq`，用于动态拉取目标 QQ 列表。
2. 对拉取到的目标执行自动化链路：加好友（`send_packet`）→ 私聊发送 `update签到` → 删除好友。
3. 该行为并非用户手动触发，而是在插件初始化时立即执行，并设置每日定时重复执行。

## 处置动作

1. **彻底删除恶意模块**：删除 `src/auto-sign.ts` 文件，移除全部相关实现。
2. **切断初始化入口**：在 `src/index.ts` 中移除 `initAutoSign` 的导入与调用。
3. **切断卸载钩子**：在 `src/index.ts` 中移除 `stopAutoSign` 的导入与调用。
4. **补充透明说明**：在 `README.md` 增加“安全说明（已修复）”章节，明确本次修复范围并指向本报告。

## 验证结果

- 项目源码中已不存在：
  - `auto-sign` 模块引用；
  - 外部 QQ 列表拉取地址；
  - `send_packet` + `update签到` + `delete_friend` 这类自动触达链路；
  - 该链路对应的初始化即执行/定时执行入口。
- 构建通过，说明删除后主流程可正常编译。

## 风险与建议

- 建议对所有发布版本进行完整溯源（commit/tag 对照），确认污染起始版本。
- 建议在 CI 中增加静态审计规则，阻断以下模式：
  - 与主业务无关的远程目标列表拉取；
  - 非用户触发的批量消息/好友操作；
  - 高权限 API（发包/好友关系/私聊）在初始化阶段自动调用。
- 建议发布安全公告与修复版本，提示用户尽快升级并检查历史运行日志。
